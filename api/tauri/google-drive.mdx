## Overview

Google Drive commands handle OAuth authentication, folder browsing, file indexing, and cloud streaming for media stored in Google Drive.

## Authentication

### connect_google_drive

Alias for `gdrive_start_auth` + `gdrive_complete_auth`. Start the OAuth flow.

<CodeGroup>
```rust Rust Signature
#[tauri::command]
async fn gdrive_start_auth() -> Result<String, String>
```

```typescript TypeScript
import { invoke } from '@tauri-apps/api/tauri';

// Step 1: Start auth flow (opens browser)
const authUrl = await invoke<string>('gdrive_start_auth');
console.log('Auth URL:', authUrl);

// Step 2: Wait for callback (user completes auth in browser)
const accountInfo = await invoke<DriveAccountInfo>('gdrive_complete_auth');
console.log('Connected:', accountInfo.email);
```
</CodeGroup>

<Info>
  **OAuth Flow**:
  1. `gdrive_start_auth` returns a Google OAuth URL and opens it in the default browser
  2. User authenticates with Google
  3. Backend server receives the code and redirects to `localhost:42069` with tokens
  4. `gdrive_complete_auth` waits for this callback, stores tokens, and returns account info
</Info>

<ResponseField name="authUrl" type="string">
  Google OAuth consent screen URL
</ResponseField>

## Folder Management

### scan_cloud_library

Alias for `gdrive_scan_folder`. Scan a Google Drive folder and index all video files.

<CodeGroup>
```rust Rust Signature
#[tauri::command]
async fn gdrive_scan_folder(
    state: State<'_, AppState>,
    window: Window,
    folder_id: String,
    folder_name: String,
) -> Result<CloudIndexResult, String>
```

```typescript TypeScript
const result = await invoke<CloudIndexResult>('gdrive_scan_folder', {
  folderId: '1abc_XYZ',
  folderName: 'My Movies'
});

console.log(`Indexed ${result.indexed_count} files (${result.movies_count} movies, ${result.tv_count} TV)`);
```
</CodeGroup>

<ParamField path="folder_id" type="string" required>
  Google Drive folder ID
</ParamField>

<ParamField path="folder_name" type="string" required>
  Folder name (for display/logging)
</ParamField>

<ResponseField name="CloudIndexResult" type="object">
  <Expandable title="Response fields">
    <ResponseField name="success" type="boolean">
      Whether scan succeeded
    </ResponseField>
    <ResponseField name="indexed_count" type="number">
      Number of new files indexed
    </ResponseField>
    <ResponseField name="skipped_count" type="number">
      Number of files already in library
    </ResponseField>
    <ResponseField name="movies_count" type="number">
      New movies added
    </ResponseField>
    <ResponseField name="tv_count" type="number">
      New TV episodes added
    </ResponseField>
    <ResponseField name="message" type="string">
      Human-readable summary
    </ResponseField>
  </Expandable>
</ResponseField>

<Info>
  **Auto-Detection**: The scanner parses filenames to detect:
  - Movies: `Movie Title (2023).mkv`
  - TV Shows: `Show Name S01E05.mkv`
  
  It fetches metadata from TMDB and downloads posters automatically.
</Info>

### get_cloud_stream_url

Alias for `gdrive_get_stream_url`. Get a temporary streaming URL for a Google Drive file.

<CodeGroup>
```rust Rust Signature
#[tauri::command]
async fn gdrive_get_stream_url(
    state: State<'_, AppState>,
    file_id: String,
) -> Result<(String, String), String>
```

```typescript TypeScript
const [streamUrl, accessToken] = await invoke<[string, string]>(
  'gdrive_get_stream_url',
  { fileId: 'abc123_file_id' }
);

// Use in video player with Authorization header
const headers = {
  'Authorization': `Bearer ${accessToken}`
};
```
</CodeGroup>

<ParamField path="file_id" type="string" required>
  Google Drive file ID
</ParamField>

<ResponseField name="tuple" type="[string, string]">
  Returns `[stream_url, access_token]`
  
  <Expandable title="Return values">
    <ResponseField name="stream_url" type="string">
      Direct download URL (valid for ~1 hour)
    </ResponseField>
    <ResponseField name="access_token" type="string">
      OAuth access token to include in `Authorization: Bearer <token>` header
    </ResponseField>
  </Expandable>
</ResponseField>

<Warning>
  Streaming URLs expire after approximately 1 hour. Access tokens must be included in all video chunk requests.
</Warning>

## Change Detection

### check_cloud_changes

Poll Google Drive for new or changed video files (uses Changes API).

<CodeGroup>
```rust Rust Signature
#[tauri::command]
async fn check_cloud_changes(
    state: State<'_, AppState>,
    window: Window,
) -> Result<CloudIndexResult, String>
```

```typescript TypeScript
const result = await invoke<CloudIndexResult>('check_cloud_changes');

if (result.indexed_count > 0) {
  console.log(`${result.indexed_count} new files added!`);
}
```
</CodeGroup>

<Info>
  **How it works**:
  1. Stores a "changes token" after each poll
  2. On next poll, fetches only files that changed since the last token
  3. Automatically indexes new video files
  4. Sends desktop notifications for new content
  5. Much more efficient than re-scanning entire folders
</Info>

<Note>
  StreamVault polls for changes every 5 seconds automatically in the background. You rarely need to call this manually.
</Note>

## Events

### cloud-scan-complete

Emitted when a folder scan finishes.

<CodeGroup>
```typescript TypeScript
import { listen } from '@tauri-apps/api/event';

await listen<{
  folder: string;
  indexed: number;
  skipped: number;
  movies: number;
  tv: number;
}>('cloud-scan-complete', (event) => {
  console.log(`Scan complete: ${event.payload.folder}`);
});
```
</CodeGroup>

---

### cloud-indexing-started

Emitted when change detection finds new files.

<CodeGroup>
```typescript TypeScript
await listen<{ count: number }>('cloud-indexing-started', (event) => {
  console.log(`Indexing ${event.payload.count} new files...`);
});
```
</CodeGroup>
